freertos源码阅读--列表和列表项

发表于2021-01-27|更新于2021-01-28
|阅读量:

列表和列表项是源码里面相对比较简单的，阅读源码就先从这里开始吧

FreeRTOS中用到了大量的列表（List）和列表项（ListItem）。

源码文件分别为list.c和list.h

相关结构体

在.h`文件定义了三个结构体：列表、列表项、迷你列表项

列表成员包括：

  • uxNumberOfItems：列表项数目。用于记录本列表有多少个列表项，每一个列表都会有一个列表尾项(xListEnd)，这个列表项不做记录数。

  • pxIndex：列表索引。注意数据类型为列表项指针，用于遍历所用列表项。

  • xListEnd：列表尾项。数据类型为迷你列表项，总是处于列表尾部。

列表项成员包括：

  • xItemValue：辅助计数值。这个用于列表项在列表中的排序。
  • pxNext：下级列表项。数据类型为列表项指针，指向下一级列表项。
  • pxPrevious：上级列表项。
  • pvOwner：指向拥有该列表项的内核对象，通常是TCB。
  • pvContainer：指向该节点所在的列表。

迷你列表项就是列表项的删减版。列表尾项数据类型就是迷你列表项。

列表头文件还有一些API，这些比较简单，后续有时间在补上。

看完头文件，再来看一下c文件。

主要包括五个函数。

列表项初始化函数

这里只有一条语句：

pxItem->pvContainer = NULL;

将列表项所在列表地址配置成NULL，表示该列表项为插入到任何列表。

列表初始化函数

输入参数为需要初始化的列表。

我们一句一句来看：

pxList->pxIndex = ( ListItem_t * ) &( pxList->xListEnd );

第一句表示将列表的列表尾项的地址(&( pxList->xListEnd ))赋值给了列表的列表索引，即列表索引指向列表尾项。

pxList->xListEnd.xItemValue = portMAX_DELAY;

第二句将列表尾项的辅助计数值设置为最大，这里表示将列表尾项排到最尾。

pxList->xListEnd.pxNext = ( ListItem_t * ) &( pxList->xListEnd );

pxList->xListEnd.pxPrevious = ( ListItem_t * ) &( pxList->xListEnd );

三四句将列表尾项的上下级指针都指向自己，由于刚初始化的列表只有一个列表尾项。

pxList->uxNumberOfItems = ( UBaseType_t ) 0U;

第五句将列表的计数值设置为0。表示没有插入列表项，列表尾项不作列表的计数。

列表尾插入列表项

函数输入参数为一个列表和一个列表项

ListItem_t * const pxIndex = pxList->pxIndex;

先定义了一个列表项指针pxIndex 保存列表的索引指针。有初始化列表函数可知，这个索引指针就是列表尾项的指针。

pxNewListItem->pxNext = pxIndex;

pxNewListItem->pxPrevious = pxIndex->pxPrevious;

上面两句是对于要插入的列表项的，将要插入的列表项的下一级指向局部变量pxIndex ，即列表尾项的指针，上一级指向列表尾项的上一级。这样就把列表项的上下级指针配置成了列表尾项的上下级指针。接下来就是把列表尾项后退一步了。

pxIndex->pxPrevious->pxNext = pxNewListItem;

pxIndex->pxPrevious = pxNewListItem;

上面两句是对于列表尾项的。列表尾项前插入了一个列表项，这里修改了列表尾项（pxIndex）的上下级指针。

最后两句，将列表项的pvContainer指向该列表，再把列表的列表项数目加1.

按升序插入列表项

输入参数为列表和列表项

ListItem_t *pxIterator;

const TickType_t xValueOfInsertion = pxNewListItem->xItemValue;

第一个局部变量用于保存列表项地址，第二个保存要插入的列表项的排序辅助值，用于遍历列表用。

既然排序插入列表项，就要对列表进行遍历了

首先将pxIterator保存列表尾项的地址

从列表尾项下一级（pxIterator->pxNext->xItemValue）开始，查找刚好排序辅助值大于的位置。

得到一个列表项，要插入的列表项的辅助值刚好大于这个列表项的辅助值，将其位置的列表项地址赋值给pxIterator，之后要做的就是在列表项pxIterator之后插入新列表项即可。于队尾插入类似。

移除列表项

输入列表项，输出列表还剩列表项数目

List_t * const pxList = ( List_t * ) pxItemToRemove->pvContainer;

保存列表项所在列表的地址pxList

pxItemToRemove->pxNext->pxPrevious = pxItemToRemove->pxPrevious;

pxItemToRemove->pxPrevious->pxNext = pxItemToRemove->pxNext;

对于要移除的列表项，将其下级于上级列表项做一下连接。

if( pxList->pxIndex == pxItemToRemove )
{
    pxList->pxIndex = pxItemToRemove->pxPrevious;
}

确保列表索引地址存在有效的列表项。

pxItemToRemove->pvContainer = NULL;
( pxList->uxNumberOfItems )--;

return pxList->uxNumberOfItems;

最后需要将列表项的pvContainer设置为NULL
